# Screenshots Ordner

Fügen Sie hier Ihre App-Screenshots ein für die GitHub README.

## Empfohlene Screenshots:

1. **dashboard.png** - Summary/Dashboard Ansicht
   - Zeigt die Übersicht mit Task-Statistiken
   - Begrüßungsnachricht
   - Upcoming Deadlines

2. **board.png** - Kanban Board mit Tasks
   - Alle vier Spalten sichtbar (ToDo, In Progress, Awaiting Feedback, Done)
   - Mindestens ein paar Tasks in verschiedenen Spalten
   - Suchleiste sichtbar

3. **task-detail.png** - Task Detail Overlay
   - Aufgeklappte Task-Detailansicht
   - Zeigt Titel, Beschreibung, Subtasks, zugewiesene Kontakte
   - Edit/Delete Buttons sichtbar

4. **contacts.png** - Kontaktliste
   - Alphabetisch sortierte Kontakte
   - Mehrere Kontakte sichtbar
   - Detail-Ansicht eines Kontakts (optional)

5. **mobile-board.png** - Mobile Ansicht des Boards
   - Vertikale Spalten-Anordnung
   - Smartphone-Format (ca. 375px Breite)

## Screenshot-Tipps:

- **Auflösung**: Mindestens 1920x1080 für Desktop, 375x812 für Mobile
- **Format**: PNG (für beste Qualität) oder JPG
- **Inhalt**: Verwenden Sie realistische Daten (keine Lorem Ipsum)
- **Dateigrö��e**: Optimieren Sie die Bilder (max. 500KB pro Screenshot)
- **Tools**: 
  - Browser DevTools für responsive Screenshots
  - Firefox Screenshot-Tool (Shift + F2, dann `screenshot --fullpage`)
  - Chrome Extension: "Full Page Screen Capture"
  - macOS: Cmd + Shift + 4 + Space
  - Windows: Win + Shift + S
  - Linux: Flameshot, GNOME Screenshot

## Wie Screenshots erstellen:

1. Öffnen Sie Ihre Join-App im Browser
2. Navigieren Sie zur gewünschten Seite
3. Füllen Sie die Seite mit realistischen Test-Daten
4. Machen Sie einen Screenshot
5. Speichern Sie ihn mit dem entsprechenden Namen in diesem Ordner
6. Optimieren Sie die Dateigröße falls nötig
